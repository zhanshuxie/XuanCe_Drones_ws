# 强化学习基本信息说明

本文档介绍两个无人机控制 Demo 的强化学习基本信息，包括低层控制器 `demo_navigate.py` 和高层规划器 `demo_simple_spread_planner.py`。

---

## 一、低层控制器 — `demo_navigate.py`

### 基本配置

| 属性 | 描述 |
|------|------|
| 算法 | PPO（单智能体） |
| 智能体数量 | 1 |
| 物理仿真 | PyBullet 3D 物理引擎（CF2X Crazyflie 无人机） |
| 并行环境数 | 10 |
| 控制频率 | 30 Hz |
| 物理频率 | 240 Hz |

### 动作空间

| 属性 | 描述 |
|------|------|
| 类型 | 连续 (Box) |
| 维度 | **4D** |
| 范围 | `[-1.0, 1.0]` |
| 语义 | `[vx, vy, vz]` 归一化速度方向 + `[speed]` 速度大小 |
| 最大速度 | 0.3 m/s |

动作通过 DSL PID 控制器转换为电机 RPM，维护最近 15 帧的动作历史缓冲。

### 观测空间（72D）

```
索引       内容                        维度
[0:3]    Roll, Pitch, Yaw（姿态角）      3D
[3:6]    Vel_x, Vel_y, Vel_z（线速度）   3D
[6:9]    AngVel_x, AngVel_y, AngVel_z  3D  （角速度）
[9:12]   Δx, Δy, Δz（到目标的相对位置）  3D
[12:72]  动作历史缓冲（4D × 15 帧）      60D
─────────────────────────────────────────
总计：72D
```

### 状态空间（20D，底层完整状态）

```
位置          (x, y, z)                          3D
四元数        (qx, qy, qz, qw)                   4D
欧拉角        (Roll, Pitch, Yaw)                  3D
线速度        (Vel_x, Vel_y, Vel_z)               3D
角速度        (AngVel_x, AngVel_y, AngVel_z)      3D
上一步动作    (a0, a1, a2, a3)                    4D
─────────────────────────────────────────────────
总计：20D
```

> 注：观测空间是状态空间的子集（去掉位置、四元数，加入相对目标位置和动作历史）。

### 奖励函数

$$r = 10.0 \times (d_{prev} - d_{curr}) - 1.0 \times d_{curr} + 100.0 \cdot \mathbf{1}[d < 0.05m]$$

| 奖励项 | 描述 |
|--------|------|
| `10.0 × (prev_dist - curr_dist)` | 接近奖励，鼓励靠近目标 |
| `-1.0 × curr_dist` | 距离惩罚，鼓励快速到达 |
| `+100.0`（一次性） | 到达目标（距离 < 0.05m）时的成功奖励 |

### 终止条件

| 条件 | 类型 |
|------|------|
| 距离目标 < 0.05m | Terminated（成功） |
| 超出边界：x/y ∈ [-2, 2]，z ∈ [0.3, 2] | Truncated |
| 倾斜过大：\|roll\| 或 \|pitch\| > 0.6 rad | Truncated |
| 超过 600 步（20 秒） | Truncated |

## 二、高层规划器 — `demo_simple_spread_planner.py`

### 基本配置

| 属性 | 描述 |
|------|------|
| 算法 | MAPPO（多智能体 PPO） |
| 智能体数量 | 3（无人机） |
| 目标点（Landmark）数量 | 3 |
| 环境类型 | 纯 2D 粒子模型（无物理仿真） |
| 并行环境数 | 16 |
| 世界坐标范围 | `[-1.0, 1.0]`（归一化） |

### 动作空间（每个智能体）

| 属性 | 描述 |
|------|------|
| 类型 | 连续 (Box) |
| 维度 | **2D** |
| 范围 | `[-1.0, 1.0]` |
| 语义 | 2D 位移方向指令（归一化） |
| 步长上限 | `max_step = 0.15`（每步实际位移上限） |

### 观测空间（每个智能体，14D）

```
索引       内容                                维度
[0:2]    own_x, own_y（自身位置，绝对坐标）     2D
[2:4]    own_vx, own_vy（自身速度/上步位移）    2D
[4:10]   到 3 个目标点的相对位置 (Δx, Δy) × 3  6D
[10:14]  到 2 个其他智能体的相对位置 × 2        4D
────────────────────────────────────────────────
总计：14D（每个智能体独立观测）
```

### 全局状态空间（18D，供 Critic 使用）

```
所有智能体位置    (x, y) × 3 agents    6D
所有智能体速度    (vx, vy) × 3 agents  6D
所有目标点位置    (x, y) × 3 landmarks 6D
────────────────────────────────────────
总计：18D（全局状态，仅 Critic 可见）
```

> MAPPO 架构：Actor 使用局部观测（14D），Critic 使用全局状态（18D）。

### 奖励函数（共享奖励，所有智能体相同）

$$r = -\sum_{l=1}^{3} \min_{i} \|pos_i - landmark_l\| - \sum_{i < j} \mathbf{1}[\|pos_i - pos_j\| < 0.15]$$

| 奖励项 | 描述 |
|--------|------|
| `-min(dist_to_landmark)` | 覆盖奖励：每个目标点由最近智能体负责，惩罚未覆盖距离 |
| `-collision_penalty (1.0)` | 碰撞惩罚：智能体间距离 < 0.15 时扣分 |

奖励为**负值域**，智能体通过最小化覆盖距离并避免碰撞来最大化奖励。

### 终止条件

| 条件 | 类型 |
|------|------|
| 超过 25 步 | Truncated（固定 Episode 长度） |
| 无自然终止条件 | — |

## 三、层级架构耦合关系

两个系统在 `demo_hierarchical_deploy.py` 中被整合为层级控制架构：

```
高层规划器 (MAPPO, 2D)               低层控制器 (PPO, 3D)
────────────────────────             ────────────────────────
每步输出 2D 位移指令                   接收 3D 航点
25 步/episode              ──→       每航点执行 32 步物理仿真
世界坐标: [-1, 1]         坐标映射    物理坐标: [-2, 2] m
                         (×scale=2.0m)  Z 轴固定 1.0m
```

**执行流程：**
1. **规划步**：MAPPO 读取当前 2D 无人机位置，输出位移动作
2. **坐标映射**：2D 位移 → 3D 航点（乘以 scale=2.0，Z 轴固定为 1.0m）
3. **导航循环**（32 步）：PPO 控制器驱动无人机飞向航点，到达后（距离 < 0.05m）悬停
4. **覆盖检测**：每个高层步结束后检查各目标点是否被覆盖（距离 < 0.2m）

### 对比总结

| 对比项 | 低层控制器 (PPO) | 高层规划器 (MAPPO) |
|--------|----------------|-------------------|
| 智能体数 | 1 | 3 |
| 环境类型 | PyBullet 3D 物理仿真 | 纯 2D 粒子模型 |
| 动作空间 | 4D 速度指令，连续 [-1,1] | 2D 位移指令，连续 [-1,1] |
| 观测维度 | 72D | 14D / agent |
| 状态维度 | 20D（底层完整状态） | 18D（全局状态） |
| Episode 长度 | 600 步（20 秒 @ 30Hz） | 25 步 |
| 奖励类型 | 稀疏+稠密，正值域 | 稠密，负值域 |
| 终止条件 | 到达目标 / 越界 / 时间 | 固定步数 |
| 任务类型 | 单机点到点导航 | 多机协作覆盖 |
